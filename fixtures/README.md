# Fixtures

Small **bundles** used by verifier tests and as examples. Each directory under `fixtures/<university>/<bundle-name>/` is a complete release directory.

## Layout

```
fixtures/uplb/sample-release/
  manifest.json           # committed; regenerated by `npm run fixtures:sync`
  courses.json
  sections.json
  schedules.json
  metadata.json
  fixture.config.json     # inputs for manifest builder + sync; may reference previous bundle for regression
```

## `fixture.config.json`

Consumed by `scripts/sync-fixture-manifests.ts` and `connectors/*/verify-release.ts`. Fields match `BuildManifestConfig` in `packages/verifier-sdk` (see `builder.ts`): `universityId`, `semesterId`, `catalogSourceId`, `connectorId`, `connectorVersion`, `trustTier`, `sourceType`, optional `previousReleaseId` / paths for regression.

## Workflow

1. Edit row JSONs (`courses.json`, …).
2. Run `npm run fixtures:sync` from repo root (`upsked-sdk/`).
3. Run `npm run test` and `npm run verify:sample` (or `npm run ci` before pushing).

Do not hand-edit `manifest.json` after sync without re-running verify.
