# Fixtures

Sample **release bundles** under `fixtures/<university>/<bundle>/` used by automated tests and as examples for connector authors.

## If you are building your own connector

- Treat **`fixtures/upb/sample-release/`** (plus `previous-release/` for regression) as the **shape** of a valid bundle, not as data you must copy.
- Your real output should live in a directory **you** control; run the verifier against it:

  ```bash
  npm run verify -- /path/to/your/bundle --previous /path/to/last-accepted
  ```

- **`sample-amis-classes-merged.json`** and **`sample-upsked-bundle/`** are **UP Baguio (UPB)** staging files for the reference pipeline in `connectors/upb/pipeline/`. Other schools will have different raw formats.

## Layout (UPB examples in this repo)

```
fixtures/upb/sample-release/
  manifest.json           # committed; regenerated by `npm run fixtures:sync`
  courses.json
  sections.json
  schedules.json
  metadata.json
  programs.json           # optional
  fixture.config.json     # inputs for manifest builder + sync; may reference previous bundle for regression

fixtures/upb/sample-upsked-bundle/
  courses.json            # AISIS-style snake_case + Upsked fields (sections-upsked, …)
  sections-upsked.json
  schedules-upsked.json
  departments.json
  metadata.json
  programs.json           # optional

fixtures/upb/sample-amis-classes-merged.json   # merged AMIS `classes` API export (seed)

fixtures/upb/full-upb-2026-1-interop/
  courses.json, sections.json, …   # large real export (camelCase); not used by CI tests
```

**CI** uses only `sample-release` (small) + `previous-release`. Do not replace `sample-release` with a full-campus export without updating tests and docs.

The **UPB connector** (`connectors/upb/pipeline/`) can produce `sample-upsked-bundle` from merged AMIS JSON; `export-interop-from-upsked.mjs` can write camelCase rows into **`full-upb-2026-1-interop/`** by default. To refresh the tiny verifier fixture, copy row JSONs into `sample-release/` and run `npm run fixtures:sync`.

## `fixture.config.json`

Used by `scripts/sync-fixture-manifests.ts` and `connectors/*/verify-release.ts`. Fields match `BuildManifestConfig` in `packages/verifier-sdk` (`builder.ts`): `universityId`, `semesterId`, `catalogSourceId`, `connectorId`, `connectorVersion`, `trustTier`, `sourceType`, optional `previousReleaseId` / paths for regression.

## Workflow (maintainers editing this repo’s fixtures)

1. Edit row JSONs (`courses.json`, …).
2. Run `npm run fixtures:sync` from repo root (`upsked-sdk/`).
3. Run `npm run test` and `npm run verify:sample` (or `npm run ci` before pushing).

Do not hand-edit `manifest.json` after sync without re-running verify.
